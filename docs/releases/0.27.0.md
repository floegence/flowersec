# Flowersec 0.27.0

Flowersec 0.27.0 makes automatic connections artifact-first, bounds connection setup work, reduces idle RPC resource use, and tightens cross-language cryptographic and logging safety. This release intentionally removes legacy connection inputs and compatibility exports; migrate callers before upgrading.

## Breaking Connection API

- Go `client.Connect` now accepts only `*protocolio.ConnectArtifact`.
- TypeScript `connect`, `connectBrowser`, and `connectNode` now accept only `ConnectArtifact`.
- Raw grants, direct connection info, readers, byte slices, serialized JSON, wrapper-shape detection, deprecated browser grant helpers, and the TypeScript `./internal` export have been removed. Explicit tunnel and direct connection APIs remain available for callers that intentionally manage those credentials.
- Browser control-plane callers must use `requestConnectArtifact` or `requestEntryConnectArtifact`. See `docs/MIGRATION_0.27.md` for the complete migration contract.
- Scoped metadata behavior is unchanged, including the existing relaxed optional-scope validation option.

## Connection Resource Bounds

- Go RPC request handlers are now dispatched lazily with the existing concurrency and queue limits instead of pre-creating a request worker pool for every stream. Queue saturation still returns the existing 429 response, and cancellation discards work that has not started.
- Go direct WebSocket handlers now expose `MaxPendingHandshakes`. The default is 256; saturated handlers reject before upgrade with HTTP 503. The bound covers upgrade, resolver work, and authentication only, and is released as soon as a session is established or setup exits.

## Security

- Rust `Debug` formatting for generated messages and artifact-related request types redacts marked tokens, pre-shared keys, and entry tickets while retaining non-sensitive diagnostic fields. Serialization and wire formats are unchanged.
- All four SDKs reject the shared non-zero low-order X25519 negative vector. Rust now explicitly rejects non-contributory shared secrets.
- Go, TypeScript, and Swift shorten the lifetime of selected decoded secrets and active record-key buffers with best-effort cleanup. This reduces sensitive-data residency but does not claim guaranteed erasure in managed or copy-on-write runtimes.
- `SECURITY.md` documents private reporting through GitHub Security Advisories or `tangjianyin@gmail.com`, supported versions, and sensitive logging guidance.

## Transport And Runtime Validation

- TypeScript encrypted-record parsing avoids unnecessary copies where cryptographic operations consume views synchronously. Swift secure-channel and Yamux receive paths use local read offsets with threshold-based compaction, without changing public buffer ownership or wire behavior.
- Chromium now exercises the complete browser-to-Go tunnel path, including P-256 setup, RPC requests and notifications, and bidirectional streams. WebKit continues to cover the Service Worker and window bridge smoke contract.
- The existing seven-cell Go-reference interoperability matrix remains the cross-language release contract, including shared positive and negative fixtures.

## Swift Package

- The Swift SDK remains a SwiftPM communication library without UIKit, SwiftUI, UIApplication, or iOS-specific application-lifecycle dependencies. Its iOS declaration records supported deployment metadata alongside macOS; the implementation and package-level quality gates are not tied to an iOS app runtime.
- AsyncHTTPClient, SwiftNIO, and NIOSSL use compatible minimum-version ranges so downstream Swift packages can resolve supported updates.
