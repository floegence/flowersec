# Flowersec 0.26.0

Flowersec 0.26.0 strengthens the communication substrate around secure connection setup, streaming proxy transport, bounded tunnel authorization, and cross-language contract enforcement. This release contains intentional breaking API removals; applications should complete the migration steps before upgrading.

## Security Defaults

- Artifact helpers now accept only absolute HTTPS URLs by default. Browser-relative TypeScript URLs are resolved against `globalThis.location` and then evaluated by the same policy; Node.js rejects relative URLs.
- Loopback HTTP requires an explicit opt-in and accepts only literal `localhost`, canonical `127.0.0.0/8`, or `::1` targets. Redirects, userinfo, unknown schemes, non-canonical loopback forms, and non-loopback HTTP are rejected.
- Rust artifact requests use `ArtifactHttpClient`, whose construction disables redirects. The raw `reqwest::Client` injection path has been removed.
- Omitted Go transport policy now means `RequireTLS`. The unrestricted `AllowPlaintext` APIs have been removed from all four SDKs; only loopback plaintext and risk-acknowledged host-scoped network plaintext policies remain.

## Proxy Transport

- TypeScript, Swift, and Rust proxy servers now stream request and response chunks as they arrive, matching the Go reference behavior and avoiding full-body aggregation.
- Proxy servers enforce an independent default limit of 64 concurrent streams and immediately reset excess inbound streams.
- Named proxy profiles have been removed. Go no longer exposes `proxy/profile`, `preset.ResolveBuiltin`, or the gateway `proxy.profile` fallback; preset manifests are the only supported preset input.
- TypeScript no longer exports `connectTunnelProxyBrowser` or `connectTunnelProxyControllerBrowser`, their grant-based option types, or duplicate artifact helper and error aliases from the browser subpath. Import artifact APIs from `@floegence/flowersec-core/controlplane`.

## Tunnel Bounds

- `MaxTokenLifetime` and `MaxInitHorizon` default to two minutes and reject signed grants outside the configured lifetime and initialization horizon.
- `MaxReplayEntries` bounds the replay cache, removes expired entries before refusing capacity, and never evicts a still-valid replay key.
- Replay keys are recorded only after external authorization succeeds and before endpoint insertion, so denied requests do not consume a token and concurrent reuse admits at most one endpoint.
- Queue accounting, channel state, and observe decisions are isolated by exact `(audience, issuer)` scope. Observe batches require scoped channel decisions and fail as a whole on missing scope, duplicates, or tenant ID mismatch.

## Connection And Quality Gates

- High-level Go connection success now means the encrypted channel and RPC bootstrap stream are established; applications can call `ProbeLiveness` when an explicit peer response check is required.
- Interoperability smoke runs all deterministic limit cases, stress coverage includes large streaming and mixed concurrent flows, and automated tests plus the loadgen checker enforce deterministic resource bounds. Machine-sensitive throughput benchmarks remain manually reviewed evidence rather than a release gate.
