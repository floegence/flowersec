# Flowersec 0.28.0

Flowersec 0.28.0 introduces the carrier-neutral Transport v2 contract, adds native QUIC and browser WebTransport alongside WebSocket, and establishes auditable weak-network and performance release gates. This release contains intentional breaking APIs; downstream applications should follow `docs/MIGRATION_TRANSPORT_V2.md` before upgrading.

## Transport v2

- Transport v2 separates admission, encrypted session establishment, logical streams, reconnect ownership, and carrier implementation behind shared contracts across Go, TypeScript, Swift, and Rust.
- WebSocket, raw QUIC, and WebTransport are equal supported carriers. Carrier selection is capability-driven and does not designate a primary or fallback protocol.
- WebSocket keeps hop-local Yamux multiplexing. Raw QUIC and WebTransport map Flowersec logical streams directly to native bidirectional QUIC streams and do not place Yamux over QUIC.
- Native QUIC stream FIN, reset, stop, flow control, stream capacity, and connection migration or UDP rebinding remain visible to the carrier implementation instead of being flattened into a byte tunnel.
- QUIC requires strict Flowersec ALPN and TLS 1.3. Transport v2 deliberately disables 0-RTT and QUIC DATAGRAM support.

## Breaking API And Lifecycle Changes

- New v2 connection flows consume bounded `ArtifactV2` sources, signed carrier capabilities, and explicit admission leases. Durable spend must be committed by the artifact owner only after the authenticated connection boundary succeeds.
- Session readiness, RPC attachment, reconnect, disconnect, stream FIN, reset, and close are asynchronous lifecycle operations. Applications must not reuse v1 `Client`, Yamux stream, or synchronous disconnect assumptions as v2 substitutes.
- Carrier cross-products and endpoint selection are represented by exact signed tuples. Applications should supply product policy and artifact acquisition without rebuilding carrier negotiation inside downstream UI or business layers.
- The v1 surfaces remain available where documented, but v1 control-plane artifacts cannot be cast or reinterpreted as Transport v2 artifacts.

## Weak-Network And Performance Evidence

- Deterministic weak-network fixtures cover latency, jitter, loss, duplication, reordering, bandwidth limits, fragmentation, black holes, UDP behavior, and cancellation or race-sensitive cleanup.
- Release evidence now has a checked-in case registry, performance manifest, runner configuration, trust policy, trust store, signed metadata schema, and fail-closed validation tooling.
- Native QUIC proofs cover independent stream IDs, sibling isolation after reset, per-stream flow-control stalls, migration or rebinding, mixed-carrier tunnel bridging, and race-sharded Go execution.
- Capacity and A/B performance gates distinguish contract-only checks, local smoke evidence, and signed release evidence so local runs cannot claim system-level results they did not measure.

## Cross-Language Support

- Go provides WebSocket, raw QUIC, and WebTransport carrier implementations plus Transport v2 admission, endpoint sets, sessions, streams, and tunnel coordination.
- TypeScript provides WebSocket and browser WebTransport connection paths, Transport v2 sessions, reconnect management, artifact leases, and browser-safe packaging.
- Rust provides native raw QUIC Transport v2 sessions and Go interoperability coverage without Yamux over QUIC.
- Swift provides the portable Transport v2 wire, cryptographic, session, stream, and lifecycle contracts while advertising only carriers implemented by the Apple runtime.
- Shared fixtures enforce strict wire, cryptographic, Unicode 15.1 IDNA, capability, artifact, endpoint-set, session, and error parity across maintained SDKs.

## Security And Release Quality

- Transport v2 binds admission, selected carrier capabilities, endpoint identity, session configuration, and cryptographic handshakes before application streams become ready.
- Sensitive artifact, token, pre-shared key, and session material remains redacted from maintained debug surfaces.
- The local release gate covers generation consistency, public API stability, coverage, race tests, vulnerability and audit checks, package validation, browser smoke, and the seven-cell Go-reference interoperability matrix.
- The release workflow validates curated notes, exact version facts, all three ecosystem tags, and signed Transport v2 evidence before publishing Go, SwiftPM, npm, and Rust artifacts.
